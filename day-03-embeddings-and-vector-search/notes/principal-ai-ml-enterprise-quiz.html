IngestionArchitecture

1. Why is enterprise document ingestion usually done offline or asynchronously?

Reveal reasoning

Live user requests should stay fast. Ingestion is heavy background work and needs retries, observability, and idempotency.

EmbeddingsIndexing

2. Why should embeddings from two different models usually not be mixed in the same vector index?

Reveal reasoning

Vector meaning is model-specific. Changing models normally means rebuilding the index or writing into a new namespace/index.

ChunkingRetrieval Quality

3. What is the main risk of chunks that are too large?

Reveal reasoning

Large chunks often retrieve irrelevant neighboring text. Principal-level systems tune chunking against retrieval metrics, not vibes.

MetadataSecurity

4. Which metadata field is most important for enterprise access control?

Reveal reasoning

Retrieval must not return chunks the user is not allowed to see. ACL metadata should be enforced before generation.

RetrievalHybrid Search

5. Why combine keyword search with semantic search?

Reveal reasoning

Keyword search is strong for exact names, IDs, and rare terms. Semantic search is strong for paraphrases and intent.

Vector DBStorage

6. What does a vector DB usually store for each chunk?

Reveal reasoning

The vector is for similarity search. The text and metadata are for context, filtering, source tracing, and governance.

QueuesReliability

7. What is the purpose of a dead letter queue?

Reveal reasoning

Retries should be bounded. Persistent failures need investigation instead of infinite retry loops.

Batch vs QueueOperations

8. When is scheduled batch ingestion a better fit than event-driven queues?

Reveal reasoning

Batch is simpler and often good enough for slow-changing corpora. Many enterprises still combine batch reconciliation with event-driven updates.

IdempotencyUpsert

9. Why should ingestion jobs be idempotent?

Reveal reasoning

Queues retry. Workers crash. Stable IDs and upsert semantics let the same job safely run more than once.

EvaluationRetrieval

10. Which metric best checks whether relevant chunks appear in the top results?

Reveal reasoning

Recall@K asks: did the correct/relevant evidence show up in the top K retrieved chunks?

RerankingQuality

11. Why use a reranker after initial retrieval?

Reveal reasoning

Initial retrieval is optimized for speed and recall. Reranking improves precision before generation.

GovernanceLineage

12. Which design helps debug a bad generated answer?

Reveal reasoning

Production systems need lineage. You should be able to explain which source chunks influenced an answer.

CleaningIngestion

13. Why clean text before embedding?

Reveal reasoning

Repeated footers, navigation, and legal boilerplate can pollute search results if embedded into every chunk.

Vector SearchScale

14. Why does a vector DB matter more as the corpus grows?

Reveal reasoning

Manual full scans teach the mechanics. Production systems need indexing, filtering, scaling, backups, and operational controls.

FreshnessDeletes

15. What should happen when a source document is deleted?

Reveal reasoning

Stale or unauthorized deleted content must not be retrieved. Deletion handling is part of production readiness.

ObservabilityOps

16. Which signals should an ingestion pipeline monitor?

Reveal reasoning

Enterprise pipelines fail in boring ways. Good monitoring catches backlog, cost spikes, and data quality issues early.

SecurityRAG

17. Where should access filtering happen in a RAG retrieval flow?

Reveal reasoning

The LLM should never receive context the user is not allowed to access. Filter early and enforce server-side.

Model ChangesRebuild

18. What usually triggers a full index rebuild?

Reveal reasoning

Any change that affects vector meaning, chunk identity, or filtering can require reprocessing the corpus.

ProductionCost

19. What is a practical way to control embedding cost?

Reveal reasoning

Incremental ingestion, hashing, dedupe, and caching are core cost controls.

System DesignPrincipal

20. Which answer best describes a production-grade ingestion design?

Reveal reasoning

Principal-level design balances quality, safety, operability, cost, and debuggability.
