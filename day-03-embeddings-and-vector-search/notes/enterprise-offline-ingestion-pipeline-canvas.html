Core Pipeline

The offline pipeline prepares clean, searchable chunks before the chat app ever receives a user question.

1

Sources

Docs, tickets, wikis, PDFs, databases, websites.

2

Extract

Pull raw files, text, timestamps, and source IDs.

3

Parse

Convert PDF, HTML, Markdown, and JSON into readable text.

4

Clean

Normalize spaces, remove boilerplate, fix noisy formatting.

5

Dedupe

Skip repeated documents and repeated chunks.

6

Metadata

Add source, type, owner, category, version, access level.

7

Chunk

Split text into meaningful pieces with stable IDs.

8

Embed

Turn each chunk into a vector using an embedding model.

9

Validate

Check empty text, missing metadata, and embedding failures.

10

Upsert

Insert new vectors, update changed vectors, delete removed ones.

Stage Notes

Each stage makes retrieval better, safer, or easier to debug.

A

Parse

Make every source look like consistent text before downstream processing.

B

Clean

Remove text noise because embeddings preserve both meaning and junk.

C

Dedupe

Avoid duplicate chunks crowding search results and wasting embedding cost.

D

Metadata

Enable filters like policy only, refund only, internal only, latest version only.

E

Validate

Catch empty chunks, wrong access labels, failed embeddings, and bad source IDs.

Upsert Shape

The vector database stores the chunk, its embedding, and metadata needed for filtering and source tracing.

const record = {
  id: "refund-policy:v3:chunk-04",
  text: "Refunds are processed within 5-7 business days.",
  embedding: [0.12, -0.44, 0.08, 0.91],
  metadata: {
    source: "refund-policy.pdf",
    source_type: "policy",
    category: "refund",
    section: "refund timeline",
    version: "v3",
    updated_at: "2026-06-20",
    access_level: "internal"
  }
};

Insert: the chunk is new and has no existing vector record.

Update: the source changed, so old chunk vectors need replacement.

Delete: the source was removed or replaced by a new version.

Do not mix: embeddings from different models in the same vector index.

Pseudo Code

This is the whole offline pipeline in implementation-shaped form.

const rawDocs = await loadFromSources();

for (const rawDoc of rawDocs) {
  const parsedText = await parseDocument(rawDoc);
  const cleanText = cleanDocumentText(parsedText);

  if (isDuplicateDocument(cleanText)) continue;

  const metadata = buildMetadata(rawDoc);
  const chunks = chunkText(cleanText, metadata);

  for (const chunk of chunks) {
    if (isDuplicateChunk(chunk.text)) continue;

    const embedding = await embed(chunk.text);

    await vectorDb.upsert({
      id: chunk.id,
      text: chunk.text,
      embedding,
      metadata: chunk.metadata
    });
  }
}
